# 04-더미데이터-시딩-가이드

## 이 스킬로 하는 일
`data.sql`에 개발/테스트용 더미 데이터를 안정적으로 추가한다.

## 적용 대상
- 신규 도메인 추가 후 샘플 데이터 필요
- 화면/기능 테스트용 기본 데이터 필요

## 입력 정보
1. 테이블 목록
2. FK 관계
3. 최소 샘플 개수
4. 테스트 시나리오별 데이터 조건

## 출력 결과
1. 테이블별 insert 구문
2. FK 순서를 고려한 데이터 세트
3. 실패/예외 시나리오용 샘플(필요 시)

## 작성 규칙
1. 이 프로젝트는 `data.sql`을 더미 데이터 중심으로 관리한다.
2. 테이블 생성 DDL은 넣지 않는다.
3. FK는 팀 규칙에 맞게 하드코딩 또는 서브쿼리 방식 중 하나로 통일한다.
4. 날짜/시간 값은 테스트 가능하도록 고정 패턴을 사용한다.

## 권장 삽입 순서
1. 부모 테이블
2. 자식 테이블
3. 교차/연결 테이블

## 주의사항
1. 샘플 데이터는 기능 시연이 가능한 수준으로 넣는다.
2. 중복 키/유니크 충돌이 없도록 값 규칙을 만든다.
3. 테스트가 실패하면 `data.sql`부터 확인한다.

## 완료 기준
1. 앱 기동 시 초기화 오류 없음
2. `./gradlew test` 통과
3. 주요 화면/API 수동 테스트 가능
