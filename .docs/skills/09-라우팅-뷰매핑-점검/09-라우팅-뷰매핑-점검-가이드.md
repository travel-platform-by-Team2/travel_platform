# 09-라우팅-뷰매핑-점검-가이드

## 이 스킬로 하는 일
컨트롤러 라우트와 Mustache 템플릿 경로를 대조해 경로 불일치/누락을 점검한다.

## 적용 대상
- 신규 페이지 추가 후
- URL 변경 후
- 템플릿 폴더 구조 변경 후

## 입력 정보
1. Controller 매핑 목록 (`@GetMapping`, `@PostMapping`)
2. View 반환 문자열 (`return "pages/..."`)
3. 템플릿 파일 목록 (`templates/pages`, `templates/partials`)

## 출력 결과
1. 라우트-뷰 매핑 표
2. 누락/오타/미연결 경로 목록
3. 수정 우선순위(치명/일반)

## 점검 항목
1. `@Controller`와 `@RestController` 사용 목적이 맞는가
2. 반환 뷰명이 실제 파일 경로와 일치하는가
3. 헤더/푸터/공통 partial include가 올바른가
4. 정적 리소스(`/css`, `/js`, `/images`) 경로가 존재하는가
5. 폼 action 경로가 실제 라우트와 연결되는가

## 작업 절차
1. Controller의 URL/뷰 반환을 수집한다.
2. 템플릿 파일 목록을 수집한다.
3. 불일치 항목을 분류한다.
4. 치명 경로(404, 렌더 실패)를 우선 수정한다.
5. 회귀 검증 후 문서에 반영한다.

## 주의사항
1. 동작 미연결(`href="#"`)은 placeholder로 명시하고 관리한다.
2. 템플릿 인코딩 문제는 추정으로 수정하지 않는다.
3. 라우트 정책은 팀 규칙(`/login-form` 등)에 맞춰 유지한다.

## 완료 기준
1. 주요 페이지 라우트가 모두 정상 렌더링된다.
2. 뷰 경로 오타/누락이 없다.
3. 정적 리소스 404가 없다.
