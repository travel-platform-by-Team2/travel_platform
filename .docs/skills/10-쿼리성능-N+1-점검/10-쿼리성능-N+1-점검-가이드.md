# 10-쿼리성능-N+1-점검-가이드

## 이 스킬로 하는 일
조회 성능을 점검하고 N+1 쿼리 가능성을 사전에 식별/개선한다.

## 적용 대상
- 목록/상세 조회 API 추가 후
- Entity 관계 확장 후
- 화면에서 연관 데이터 다건 표시 시

## 입력 정보
1. 대상 조회 API 목록
2. Entity 관계 구조
3. 실제 쿼리 로그
4. 응답 시간/건수

## 출력 결과
1. 성능 점검 리포트
2. N+1 의심 지점 목록
3. 개선안(쿼리/매핑/페이징)

## 점검 항목
1. 연관 엔티티 LAZY/EAGER 설정 적절성
2. 목록 조회 시 페이징 적용 여부
3. 불필요한 연관 로딩 여부
4. 동일 API 호출 시 쿼리 수 과다 여부
5. 정렬/검색 조건에서 인덱스 고려 여부

## 개선 방향 예시
1. JPQL fetch join 또는 필요한 필드만 조회
2. 목록 조회는 DTO projection 우선 고려
3. 화면 요구 데이터만 응답하도록 축소
4. 대량 조회는 페이지네이션 강제

## 주의사항
1. 성능 개선 전에 측정 근거를 남긴다.
2. 추정 기반 리팩터링을 금지한다.
3. 개선 후 기능 회귀 테스트를 반드시 수행한다.

## 완료 기준
1. N+1 의심 쿼리 제거 또는 원인 명확화
2. 주요 조회 API 성능 기준 충족
3. 변경 사항 문서화 완료
