# 02-버전-분할-설계-가이드

## 이 스킬로 하는 일
큰 기능을 V1, V2, V3으로 분해해서 팀원이 병렬로 작업하기 쉽게 만든다.

## 적용 대상
- 기능이 크거나 복잡한 경우
- 화면/API/도메인이 동시에 변경되는 경우
- 담당자 분리가 필요한 경우

## 입력 정보
1. 기능 주제
2. 현재 완료 범위
3. 남은 요구사항
4. 제외해야 할 범위

## 출력 결과
1. 버전별 문서(마크다운)
2. 각 버전의 개발 범위/제외 범위
3. 각 버전의 완료 기준

## 버전 분할 원칙
1. 한 버전은 한 가지 핵심 목표만 가진다.
2. 읽기 전용 -> 쓰기 기능 -> 안정화 순으로 나눈다.
3. 인증/권한/예외 처리 시점도 버전에 포함한다.
4. 다음 버전 의존 조건을 문서에 명시한다.

## 권장 템플릿
1. 버전 이름
2. 개발 범위
3. 제외 범위
4. 완료 기준
5. 리스크 메모(선택)

## 주의사항
1. 완료된 작업을 과대평가하지 않는다.
2. 버전 간 중복 범위를 최소화한다.
3. “나중에” 같은 모호한 표현 대신 실제 작업 단위로 작성한다.

## 완료 기준
1. 팀원이 문서만 보고 바로 착수 가능
2. 버전마다 범위가 겹치지 않음
3. 노션 업로드 시 구조가 깨지지 않음
