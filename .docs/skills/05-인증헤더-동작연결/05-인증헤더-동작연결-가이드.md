# 05-인증헤더-동작연결-가이드

## 이 스킬로 하는 일
로그인/로그아웃/회원가입 흐름과 헤더 상태 분기 동작을 연결한다.

## 적용 대상
- 인증 기능 신규 연결
- 헤더 로그인 상태 분기 구현
- 인증 라우트 정리

## 입력 정보
1. 인증 관련 URL 목록
2. 템플릿 경로(`pages/login`, `pages/signup`, `pages/main-index` 등)
3. 세션 키 이름
4. 성공/실패 리다이렉트 정책

## 출력 결과
1. 인증 컨트롤러 라우트 정리
2. 세션 상태 기반 헤더 분기 처리
3. 인증 실패 처리 규칙

## 권장 작업 순서
1. 페이지 라우팅 정리(GET)
2. 회원가입 처리(POST /join)
3. 로그인 처리(POST /login)
4. 로그아웃 처리(GET /logout)
5. 헤더 분기(로그인/비로그인)

## 구현 규칙
1. 로그인 성공 시 세션 사용자 저장
2. 로그아웃 시 세션 무효화
3. 인증 실패 메시지와 리다이렉트 경로를 분리 관리
4. 헤더 partial에서 상태별 메뉴를 명확히 분기

## 주의사항
1. `@Controller`와 `@RestController` 용도를 혼동하지 않는다.
2. 뷰 경로와 실제 템플릿 파일명을 반드시 대조한다.
3. 임시 하드코딩 세션값 사용 시 TODO로 명시한다.

## 완료 기준
1. 로그인/로그아웃/회원가입 흐름이 끊김 없이 동작
2. 헤더 상태가 세션 변경 직후 반영
3. 인증 관련 회귀 테스트 통과
