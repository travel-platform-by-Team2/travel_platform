# 14-자바-코드컨벤션-적용-가이드

## 이 스킬로 하는 일
Java 코드 생성/수정 작업에서 `.docs/specs/java-code-convention.md`를 강제 적용해 컨벤션 일관성을 유지한다.

## 적용 대상
- 엔티티/리포지토리/서비스/컨트롤러/DTO 생성
- 새 기능/도메인/엔드포인트 추가
- 기존 Java 코드의 컨벤션 위반 수정

## 참조 문서
- `.docs/specs/java-code-convention.md`
- `.docs/RULES.md`

## 입력 정보
1. 작업 목표(생성/수정 대상 클래스)
2. 도메인명과 패키지 경로
3. View/API 여부와 컨트롤러 분리 필요 여부
4. API 입력/출력 형태(JSON 요청 본문, 응답 DTO)
5. 승인 필요 작업 여부(RULES 기준)

## 출력 결과
1. 컨벤션 준수 코드(클래스/메서드/DTO/매핑)
2. 적용 규칙 체크리스트 결과
3. 변경 파일 목록
4. 검증 결과(컴파일/테스트)

## 작업 절차
1. 작업 시작 전에 `java-code-convention.md`의 해당 섹션(엔티티/서비스/컨트롤러/DTO)을 확인한다.
2. View 컨트롤러와 API 컨트롤러를 분리하고 명명 규칙을 적용한다.
3. API 입력/출력 규칙을 적용한다(`@RequestBody` 요청 본문 전용, `GET` 미사용, DTO 반환).
4. 서비스 레이어에서 DTO 변환을 처리하고 Controller에 Entity 노출을 금지한다.
5. 검증 후 결과를 보고 형식으로 정리한다.

## 주의사항
1. 컨벤션 문서와 충돌하는 임의 스타일 적용 금지.
2. 컨벤션 미정 항목은 추정하지 말고 문서 갱신 또는 사용자 확인 후 진행.
3. 대규모 변경은 승인 기준을 먼저 확인한다.

## 완료 기준
1. 변경 코드가 `.docs/specs/java-code-convention.md` 규칙과 일치한다.
2. View/API 컨트롤러 분리 및 `/api` 규칙 위반이 없다.
3. `@RequestBody`/`@RequestParam`/`@PathVariable` 사용 규칙 위반이 없다.
4. 컴파일 또는 테스트 검증이 성공한다.
