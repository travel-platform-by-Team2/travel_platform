# 06-문서동기화-업데이트-가이드

## 이 스킬로 하는 일
코드 변경 후 운영 문서와 상태 문서를 최신 상태로 맞춘다.

## 적용 대상
- 기능 추가/수정 완료 후
- 구조/정책 변경 후
- 팀 인계 전

## 입력 정보
1. 변경 파일 목록
2. 변경 이유
3. 검증 결과(명령/결과)
4. 남은 TODO/리스크

## 출력 결과
1. 작업 로그 업데이트
2. 상태 문서 업데이트
3. 정책 문서 반영(필요 시)

## 업데이트 대상 권장 순서
1. `.docs/.task/.{작업자폴더}/WORKLOG.md`
2. `.docs/.task/.{작업자폴더}/flow/*` 관련 작업 문서
3. `.docs/RULES.md` (규칙 변경 시)

## 작성 규칙
1. 사실 기반으로 작성한다.
2. 완료/진행중/예정을 구분한다.
3. 추정 표현보다 근거(테스트, 로그)를 우선한다.
4. 문서만 갱신하고 코드 변경이 없으면 그 사실을 명시한다.

## 주의사항
1. 문서와 코드가 불일치하면 문서를 즉시 수정한다.
2. 상태 문서는 요약 중심으로 유지한다.
3. 과거 이력 덮어쓰기보다 누적 기록을 권장한다.

## 완료 기준
1. 팀원이 문서만 읽고 현재 상태를 파악 가능
2. 변경 이유/결과/리스크가 문서에 모두 반영
3. 주요 운영 문서 간 내용 충돌 없음
