# 11-권한-접근제어-가이드

## 이 스킬로 하는 일
세션 사용자 기준으로 본인 데이터만 접근하도록 권한 검증 규칙을 설계/적용한다.

## 적용 대상
- 예약, 일정, 게시글 등 사용자 소유 데이터
- 수정/삭제/상세 조회 API
- 인증 필수 페이지/기능

## 입력 정보
1. 보호 대상 리소스 목록
2. 소유자 식별 필드(예: `user_id`)
3. 인증 방식(세션)
4. 접근 허용/차단 정책

## 출력 결과
1. 권한 검증 규칙 표
2. 서비스 레이어 권한 체크 포인트
3. 인증 예외 URL 목록
4. 권한 실패 응답 정책(401/403)

## 기본 원칙
1. 최종 권한 검증은 Service에서 보장한다.
2. Controller는 인증 컨텍스트 전달만 담당한다.
3. 본인 리소스 접근이 아니면 차단한다.
4. 차단 시 이유에 맞는 상태 코드(401/403)를 반환한다.

## 작업 절차
1. 리소스별 소유자 기준을 정의한다.
2. CRUD 별 권한 규칙을 정의한다.
3. 서비스 메서드에 권한 체크를 삽입한다.
4. 필터/인터셉터와 서비스 검증 역할을 분리한다.
5. 권한 실패 테스트를 추가한다.

## 주의사항
1. 필터에서만 권한을 끝내지 않는다(서비스에서 재검증).
2. 인증/권한 실패를 200으로 숨기지 않는다.
3. 디버그 로그에 민감 정보가 노출되지 않게 한다.

## 완료 기준
1. 본인/타인 접근 케이스가 명확히 분기된다.
2. 권한 실패 시 상태 코드와 메시지가 일관된다.
3. 권한 회귀 테스트가 통과한다.
