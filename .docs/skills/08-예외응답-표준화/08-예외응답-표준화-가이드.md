# 08-예외응답-표준화-가이드

## 이 스킬로 하는 일
예외 처리 규칙을 통일해서 API/화면에서 일관된 오류 응답과 메시지를 제공한다.

## 적용 대상
- 신규 API 개발
- 기존 예외 처리 리팩터링
- 오류 코드/메시지 정리 작업

## 입력 정보
1. 도메인별 실패 시나리오
2. 현재 예외 클래스 구조
3. HTTP 상태 코드 매핑 규칙
4. 사용자 노출 메시지 정책

## 출력 결과
1. 예외 클래스 정리안
2. 상태 코드 매핑 표
3. 전역 예외 처리 규칙
4. 화면 리다이렉트/응답 메시지 규칙

## 기본 매핑 규칙
- 400: 입력값 오류, 검증 실패, 중복
- 401: 인증 실패 또는 미인증
- 403: 권한 부족
- 404: 리소스 없음
- 409: 상태 충돌
- 500: 서버 내부 오류

## 작업 절차
1. 현재 예외 발생 지점을 수집한다.
2. 예외를 도메인 단위로 분류한다.
3. 상태 코드 매핑을 표준으로 확정한다.
4. GlobalExceptionHandler 응답 형식을 통일한다.
5. UI 메시지와 로그 메시지를 분리한다.

## 주의사항
1. 사용자 메시지에 내부 시스템 정보를 노출하지 않는다.
2. 화면 리다이렉트와 JSON 응답을 혼용하지 않는다.
3. 동일 원인인데 상태 코드가 다르게 나가지 않도록 통일한다.

## 완료 기준
1. 주요 실패 시나리오가 표준 상태 코드로 응답한다.
2. 테스트에서 예외 응답 포맷이 일관된다.
3. 로그와 사용자 메시지가 목적에 맞게 분리된다.
