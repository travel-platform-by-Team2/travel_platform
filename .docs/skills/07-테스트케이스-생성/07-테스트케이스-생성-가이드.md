# 07-테스트케이스-생성-가이드

## 이 스킬로 하는 일
기능별 필수 테스트 시나리오를 만들고, 코드로 바로 옮길 수 있는 형태로 정리한다.

## 적용 대상
- 신규 기능 개발 직후
- 버그 수정 후 회귀 검증 필요 시
- 릴리스 전 안정화 단계

## 입력 정보
1. 기능 요구사항
2. API 스펙
3. 권한/인증 정책
4. 예외 처리 정책

## 출력 결과
1. 테스트 시나리오 목록
2. 단위/통합 테스트 초안
3. 실행 결과 기록 템플릿

## 최소 시나리오 세트
1. 정상 케이스 1개 이상
2. 검증 실패 케이스 1개 이상
3. 권한 실패 케이스 1개 이상(해당 시)
4. 예외 처리 케이스 1개 이상

## 권장 구조
1. Given: 데이터/세션/요청 조건
2. When: API 호출 또는 서비스 메서드 실행
3. Then: 상태 코드/응답/DB 상태 검증

## 구현 규칙
1. 테스트명은 시나리오가 드러나게 작성한다.
2. Arrange/Act/Assert 구간을 분리한다.
3. 하드코딩 값은 의미 있는 샘플로 사용한다.
4. 실패 케이스 메시지까지 검증한다.

## 주의사항
1. 테스트 없는 핵심 로직 변경 금지.
2. flaky 테스트를 만들지 않는다.
3. 테스트가 통과해도 스펙과 불일치하면 실패로 본다.

## 완료 기준
1. 최소 시나리오 세트 충족
2. `./gradlew test` 통과
3. 회귀 검증 관점에서 누락 케이스 없음
