# TASK12 - ChatbotService LLM-driven flow refactor

## 0. Work Type

- Refactoring

## 1. Goal

- Move chatbot decision flow from rule-based service logic to LLM-driven logic.
- Let LLM handle intent/SQL/answer generation.

## 2. Scope

- Refactor `ChatbotService` orchestration logic
- Extend `ChatbotLlmClient` contract for planning and DB-answer generation
- Update `OpenAiChatbotLlmClient` implementation
- Update `ChatbotServiceTest`
- Sync progress/worklog docs

## 3. Out of Scope

- Auth/session policy changes
- SQL safety policy changes
- API endpoint/signature changes
- Frontend changes

## 4. Done Criteria

- [x] `ChatbotService` no longer uses keyword-based classification or hardcoded SQL plans.
- [x] LLM returns plan (`needsDb/queryIntent/querySummary/sql/answer`) and service follows it.
- [x] DB path final answer is generated by LLM using query result rows.
- [x] `./gradlew test --tests "*ChatbotServiceTest"` passes.

## 5. Risk

- Level: MEDIUM
- Reason: behavior change in core chatbot decision flow
