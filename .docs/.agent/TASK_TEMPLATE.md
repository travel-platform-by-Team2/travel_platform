# TASK_TEMPLATE.md

이 문서는 TASK 문서를 빠르게 작성하기 위한 표준 템플릿이다.
세부 규칙은 `.docs/RULES.md`, `TASK_POLICY.md`를 따른다.

관련 문서:

- `.docs/.agent/SKILL_OPERATING_GUIDE.md`
- `.docs/.agent/WORKLOG_POLICY.md`

---

## 사용 규칙

1. TASK는 하나의 목표만 가진다.
2. 범위 외 작업은 포함하지 않는다.
3. 위험도는 `LOW / MEDIUM / HIGH` 중 하나로 명시한다.
4. HIGH 위험도는 사용자 승인 후 진행한다.
5. TASK 종료 시 개인 작업일지(`.docs/.task/.{개인폴더명}/TASK_WORKLOG.md`)에 결과를 기록한다.

---

## 파일명 규칙

1. 경로: `.project/.TASK/.{개인폴더명}`
2. 파일명: `TASK01.md`, `TASK02.md`, `TASK03.md` ...
3. 번호는 재사용하지 않는다.
4. 각자 본인 폴더만 사용한다.

예시:

```text
.project/.TASK/.khj/TASK01.md
```

## 작성 예시

```markdown
# TASK07 - 게시글 수정 API 검증 보강

## 0. 작업 유형

- 테스트 보강

## 1. 작업 목표

- 게시글 수정 API의 검증 실패 케이스 테스트를 추가한다.

## 2. 작업 범위

- post 서비스 통합 테스트 코드 추가

## 3. 작업 제외 범위

- Entity/DB schema 변경
- 컨트롤러 시그니처 변경

## 4. 완료 기준

- [ ] 검증 실패 케이스 2건 이상 추가
- [ ] `./gradlew test` 통과

## 5. 예상 영향 범위

- 대상 레이어: Test(Service Integration)
- 대상 기능: 게시글 수정
- 사용자 영향: 없음(내부 품질 개선)

## 6. 위험도

- 등급: LOW
- 근거: 운영 코드 변경 없음

## 7. 승인 필요 여부

- 필요 여부: 아니오
- 승인 필요 사유: 해당 없음

## 8. 검증 계획

- 빌드/컴파일: `./gradlew testClasses`
- 테스트 명령: `./gradlew test --tests *Post*`
- 수동 검증 항목: 없음

## 9. 결과 기록(작업 후 작성)

- 변경 파일: (작업 후 기입)
- 테스트 결과: (작업 후 기입)
- 미완료 항목(TODO): (작업 후 기입)
- 리스크/주의사항: (작업 후 기입)
```
