## 작업 내역

### 1. `map-detail.mustache` 반응형 검색 UI 개편
- 상단 검색 영역을 반응형 카드형 레이아웃으로 재구성함.
- 검색 필드를 다음 구조로 정리함.
  - `광역시/도`
  - `가는 날`
  - `오는 날`
  - `인원수`
  - `탐색하기` 버튼
- 브레이크포인트별 동작을 추가함.
  - 작은 화면: 2행 구조 유지
  - 매우 작은 화면: 1열 스택 전환
  - 넓은 화면: 1행 배치 전환
- 확대 배율에서 버튼이 떠 보이던 문제를 막기 위해 버튼을 별도 액션 래퍼(`map-detail-search-action`)로 감싸고 하단 정렬을 고정함.
- 모바일 지도 화면에서 좌측 결과 패널 폭을 줄여 지도 가림을 완화함.

### 2. `map-detail.js` 검색 상태 복원 및 자동 검색
- URL 쿼리스트링에서 아래 값을 읽어 검색 폼에 복원하는 로직 추가.
  - `region`
  - `checkIn`
  - `checkOut`
  - `guests`
- 쿼리스트링이 있는 경우 페이지 진입 직후 `탐색하기`가 자동 실행되도록 초기 검색 트리거를 추가함.

### 3. `booking-complete.mustache` 탐색 복귀 링크 개선
- `탐색으로 돌아가기` 링크를 `/bookings/map-detail`로 연결.
- 링크에 검색 상태를 유지하기 위한 쿼리스트링을 포함하도록 변경.
  - `region={{regionKey}}`
  - `checkIn={{checkIn}}`
  - `checkOut={{checkOut}}`
  - `guests={{guests}}`

### 4. `BookingController.java` 완료 페이지용 지역 키 계산 추가
- 완료 페이지 렌더링 시 `regionKey`를 모델에 추가함.
- 한글 지역명에서 `map-detail`에서 사용하는 region key로 변환하는 `normalizeRegionKey` 메서드를 추가함.
  - `서울 -> seoul`
  - `부산 -> busan`
  - `제주 -> jeju`
  - `경주 -> gyeongju`
  - `강원 -> gangwon`
- 매칭 실패 시 기본값은 `busan`으로 처리함.

### 5. `header-logged-in.mustache` 브랜드 링크 수정
- 로그인 상태 헤더의 `TravelMate` 브랜드 링크를 `#`에서 `/`로 변경함.
- 클릭 시 메인 페이지(`pages/main-index`)로 이동하도록 수정함.

## 의도
- 예약 완료 후 `map-detail` 복귀 시 사용자가 보고 있던 검색 조건을 유지하기 위함.
- 모바일/축소 환경에서 `map-detail` 상단 검색 UI가 더 효율적으로 보이도록 하기 위함.
- 지도 화면에서 결과 패널이 과도하게 공간을 차지하는 문제를 줄이기 위함.

## 주의사항
- 검색 상태 복원은 쿼리스트링 기반이므로, 링크를 직접 열 때 파라미터가 없으면 기본값(`busan`)이 적용될 수 있음.
- Mustache 템플릿에서 URL 파라미터 인코딩은 별도 처리하지 않았으므로, 현재는 기존 포맷(`성인 2명`, `YYYY-MM-DD`) 기준으로 사용함.
